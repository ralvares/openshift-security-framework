OpenShift Network Policy

This page provides quick access to the main documentation topics for OpenShift Network Policies. Each section below includes a link and a brief overview of the content, focused on cluster, admin, and developer network policy controls and workflows.

Admin Network Policies

Comprehensive guide to the AdminNetworkPolicy (ANP) and BaselineAdminNetworkPolicy (BANP) model in OpenShift. Explains the cluster perimeter, default-deny posture, how admin and developer policies interact, and provides examples and best practices for enforcing network security at scale.

ExternalName Service Egress Control

Describes how to securely enable egress to external services (such as APIs) using the ExternalName pattern. Focuses on using AdminNetworkPolicy to allow egress only for approved namespaces, with clear, auditable controls for external connectivity.

Onboarding Workflow

Step-by-step onboarding process for application teams and administrators. Covers namespace preparation, policy enforcement, requesting exceptions (like kube-API or corporate egress), and explains the developer experience under the enforced security model.

Operationalization Plan

Outlines the phased approach to rolling out and governing network security controls in OpenShift. Details objectives, principles, governance lifecycle, and how to maintain and evolve the AdminNetworkPolicy suite over time.
